Ontario Infrastructure Clustering Project

Python clustering analysis and Power BI dashboard for Ontario public infrastructure project data.

Project summary: This project analyzed Ontario public infrastructure projects using Python-based data cleaning/visualization (Also cleaned to double check by Power Query), feature engineering, hierarchical clustering, and Power BI dashboard visualization. The goal was to identify patterns across project category, project status, funding sources, budget, completion year, and location.

View Python notebook View Power BI dashboard file Watch dashboard demo video

Project Goal

The project uses clustering to group Ontario infrastructure projects with similar characteristics. Instead of only summarizing projects by category or status, the analysis combines multiple features, including funding type, project budget, completion year, location, and project category, to uncover broader patterns in public infrastructure investment.

Data and Features

The dataset contains Ontario infrastructure project records. The analysis selected variables that were useful for clustering and visualization while removing columns that introduced excessive noise, null value, or textual detail that was less useful for this clustering workflow.

Feature Group Variables Used Purpose
Project identity Project ID Track individual infrastructure projects.
Project type Category Compare communities, education, roads and bridges, recreation, child care, health care, and transit projects.
Project progress Status, target completion year Identify patterns between complete, planning, and under-construction projects.
Financial information Estimated total budget, municipal/provincial/federal/other funding flags Analyze project size and funding structure.
Spatial information Latitude, longitude Support geographic visualization and GIS mapping.

Python Workflow

The Python notebook follows a complete applied data analysis workflow, starting from raw data cleaning and ending with cluster output preparation for Power BI.

  • Loaded Ontario infrastructure project data into Python.
  • Dropped columns that were too noisy, sparse, or not useful for the clustering objective.
  • Filtered missing or invalid project status, latitude, and longitude values.
  • Cleaned and standardized project completion dates into a target completion year.
  • Converted funding source columns into binary indicators.
  • Performed exploratory checks on category, status, funding, location, and budget variables.
  • Removed outliers before final clustering.
  • Used Gower distance to handle mixed data types.
  • Compared complete and average linkage hierarchical clustering.
  • Selected a final cluster solution and exported the clustered dataset for Power BI.

Clustering Method

The project used hierarchical agglomerative clustering with Gower distance because the dataset contains mixed variable types, including categorical project categories, binary funding flags, numeric budgets, completion years, and geographic coordinates. Complete linkage and average linkage were compared using silhouette scores.

Gower distance Used to support mixed numerical, categorical, and binary project features.
Average linkage Selected after comparing clustering performance and interpretability.
5 clusters Final cluster solution chosen for interpretability.
Power BI output Clustered data exported for dashboard visualization.

Final Cluster Sizes

The final hierarchical clustering solution grouped projects into five clusters. The clusters were then interpreted using project counts, funding patterns, project categories, status, and average budget.

Cluster Number of Projects General Interpretation
Cluster 1 345 Smaller cluster with distinct project/funding characteristics.
Cluster 2 1,948 Largest group, representing a major common pattern among projects.
Cluster 3 1,216 Second-largest group with another common infrastructure profile.
Cluster 4 229 Smaller specialized project group.
Cluster 5 452 Moderate-sized project group with distinguishable funding/category structure.

Key Findings

  • Education, health care, and child care projects were often associated with other funding and tended to have higher budgets.
  • Community projects generally had lower budgets and commonly received both federal and municipal funding.
  • Child care and community projects were mostly complete in the cleaned project dataset.
  • Incomplete projects tended to have higher budgets, which is reasonable because large projects may take longer to plan and complete.
  • A larger number of clusters could reveal more detailed patterns, but the five-cluster solution was more interpretable for dashboard communication.

Power BI Dashboard

The clustered dataset was exported into Power BI to create an interactive dashboard. The dashboard was designed to let viewers explore project clusters, categories, funding sources, status, location, and budget patterns visually.

The dashboard complements the Python notebook by turning the clustering output into a communication-focused project that is easier to explore and present.
